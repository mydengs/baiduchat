工具调用兼容层

工具调用配置

控制 OpenAI tools / function calling 的解析、流式缓冲、失败兜底和客户端适配策略。普通聊天仍可流式输出；工具调用建议先缓冲成完整结构后再返回给客户端执行。

协议策略

不同客户端对工具调用的容忍度不同。Cline、Chatbox 通常更依赖标准 OpenAI tool_calls；Cherry Studio、Tokeny 等 Agent 客户端可能出现 DSML 文本，需要额外解析和兜底。

工具保护规则

这些规则只用于防止明显异常的工具参数污染客户端。当前项目不再使用隐藏的业务关键词拦截，不会因为路径包含“正文”“第05章”等正常词语而阻止工具调用。

推荐配置

Cherry Studioauto + cherry兼容 DSML 兜底和流式缓冲。
Clineforce_buffer + cline优先标准 OpenAI tool_calls，避免半截流式工具调用。
Chatboxauto + chatbox标准 tools 优先，失败时清理文本。
Hermes Agentauto + hermes预留 XML/结构化兜底适配。
Tokenyauto + tokeny适合多轮 Agent 工具执行，保持缓冲解析并允许工具结果后继续下一步工具。
